注释之间不能进行嵌套

属性可以写在开始标签或者自结束标签中，还可以设置属性，属性是个键值对的形式，具体的属性名和属性值取决于文档的编程规范，其中有些属性是没有值的，键值对之间使用空格进行分隔

文档声明(doctype)用于告诉浏览器当前网页的版本，位于第一行 例如： 不区分大小写

meta标签用来设置网页的元数据可以通过meta标签来设置网页的字符集，避免出现乱码问题，meta标签写在head中

html是文件的根标签，网页中的所有元素都要出现在根元素的里面

head是网页的头部，head中的内容不会在网页中直接出现，主要用来帮助浏览器或搜索引擎来解析网页

title中的内容会显示在浏览器的标题栏，搜索引擎会根据title中的内容来判断页面的主要内容

body是html的子元素，表示网页的主题，网页中所有的可见内容都该放在body中，其中所有的内容都用户可见

例如空格或者< 、>该类的符号会被浏览器解析为别的意思，故需要使用实体来进行表示

使用meta中的属性进行命名时，注意对应的变量名是固定好的，而并不是自己随意添加的

lang元素指的是使用的语言

在网页中html专门用来负责页面的结构，所以在使用html标签时，应该关注的是标签的语义而不是标签的格式

不同的标题等级表示的是重要级，一般来说h1在网页中的重要性仅次于title，一般来说一个页面只会有一个h1，由于一般使用标题标签只会最多用到h3，剩下的很少用，因为其标注的重要性太低了

在页面中独占一行的元素称之为块元素(block element)

在页面中不会独占一行的元素称之为行内元素(inline element)

hgroup指的是标题组，用来为标签分组，可以将一组相关的标题放入同一个标题组

em标签用于表示语音语调的一个加重

strong表示的是强调的重要内容

q表示是一个短引用

blockquote是一个长引用

在网页中一般通过块元素对页面进行宏观的一些布局，而行内元素是主要用来包裹文字来实现特殊的效果，一般是在块元素中放置行内元素，而不会在行内元素中放块元素，并且p元素中不能放任何块元素

浏览器会在解析网页的时候，会自动对网页中不符合规范的内容进行修正，比如：将标签写在了根元素的外部、p元素中嵌套了块元素、根元素中出现了除了head和body以外的子元素

header表示网页的头部(可以有多个头部)

main表示网页的主题部分(一个页面中只会有一个main)

footer表示网页的底部

nav表示网页的导航

aside表示和网页主题相关的其他内容(侧边栏)

article表示的是一个独立的文章

section表示的是独立的区块，当以上的标签没办法合理的表示时，用section

div没有语义，就用来表示的是一个区块，目前往往还是div是页面的主要布局元素

span没有语义，是一个行内元素，一般用于网页中选中文字

ul表示的创建无序列表，使用li来表示列表项

ol表示的创建有序列表，用li表示列表项

dl表示定义列表，使用dt来表示定义的内容，使用dd来对内容进行解释说明

最常使用的是ul标签，并且列表之间可以互相嵌套

超链接也是一个行内元素，在a标签中可以嵌套除了它自身类型以外的任意元素

超链接的href可以指定外部的网站的地址，也可以写一个内部页面的地址，当我们需要跳转到一个服务器内部的页面时，一般我们都会使用相对路径，相对路径都会使用./或者../开头，如果没有以这两个开头，则会默认添加一个./

  • ./ 表示的是文件所在目录

  • ../ 表示的是当前文件所在目录的上一级目录

./inner/123.html表示的是返回所在文件的目录下寻找inner文件夹进一步寻找123.html文件，所有文件都可以通过这样的方法往上往下寻找到的

可以将超链接的href属性设置为只有一个#，这样点击这个超链接以后，页面不会进行跳转，而是转到当前页面的顶部位置

id表示的是该元素的特定标识，每一个标签都可以添加一个id属性，同一个页面不能出现重复的id属性，id属性是字母开头

img元素中的src属性指定的是外部图片的路径，这个路径可以是你存储的图片位置（路径的调用规则是一样的），也可以是网上图片的网络地址，img的alt属性表示的是对图片的描述，这个描述是在默认情况下不会显示，有些浏览器会在图片无法加载时显示，搜索引擎会根据这个属性进行识别图片

img属性中的width和height属性的单位是像素，如果只是改变两个其中一个则另外属性会跟着等比例变化，但是如果两个属性都赋值改变了则根据你赋值的值进行缩放，在pc端往往不改变图片的这两个属性，美工会裁剪好使用的图片，减少性能开销

常见的图片格式：

  • jpeg(jpg)：支持的颜色比较丰富，不支持透明效果，不支持动图，一般用来显示照片
  • gif：支持的颜色比较少，支持简单透明和动图，一般用于单一的图片、动图
  • png:支持的颜色丰富，支持复杂透明，不支持动图，往往用网页部分
  • webp：该格式是谷歌新推出的专门用于表示页面中的图片格式，它具备其他三种格式的所有优点，而且文件还特别的小，但是缺点是其兼容性不好，有些老浏览器没办法兼容
  • base64：将图片使用base64编码，这样可以将图片转化成字符来引用页面中，一般是一些需要和页面一起加载的图片才会使用base64（直接将这个编码复制到img元素中的src内中）

iframe表示创建一个内联框架，相当于在当前页面内创建了一个小分块来显示外部网页，其中也是使用src进行标注要显示的页面，可以通过改变iframe标签的属性来改变分块的大小

audio标签用于向页面引入一个外部的音频文件，音视频文件引入时默认情况不允许用户自己播放停止

audio属性：

  • controls 是否允许用户控制播放(标注即可无需赋值)
  • autoplay音频文件是否自动播放(标注即可无需赋值)
    • 如果设置了autoplay则音乐会在打开文件的时候自动播放，但是目前大部分情况下是不允许其自动播放的
  • loop属性表示的是循环播放(标注即可无需赋值)

内联样式相当于将html和css复合起来了，使得其耦合，这样的样式不合理

由于id具有独特性，不能有重复的元素，所以html中有class来保证相似元素的类型设置，对应的有id选择器和class选择器，一般在开发中往往使用class选择器，同时可以对一个元素指定多个class，多个class之间使用空格隔开

选择器分类：元素选择器、id选择器、类选择器、通配选择器（所有元素都用这个格式）

块元素也有对应的title属性，当光标放置在这个属性上时，会显示出来对应的title

通过css添加的文字无法用光标进行选择

由于内联样式的设定的优先级最高，所以一般不建议使用内联样式去决定元素的样式，一旦使用内联样式去指定元素的样式会使得该元素无法被修改

当样式编写后没有被生效，可能是因为选择器的权重没有被记录

当子元素相加的元素加起来不等于父元素内容区的宽度，则称之为过度约束，即等式会自动进调整使得等式保持满足

如果父元素没有设置高度，则其高度取决于子元素的高度大小（子元素将父元素所撑开）

如果父元素设置了高度，则父元素的高度取决于其自身所定义的高度

可以使用overflow属性来设置父元素如何处理溢出的子元素

相邻的垂直方向外边距会发生重叠现象，兄弟元素之间的响铃垂直外边距会取两者之间的较大值（两者都是正值，如果相邻的外边距一正一负，则取二者的和，如果相邻的外边距都是负值，则取两者中绝对值较大的）

display可以用来设置成元素显示的类型

在开发中由于浏览器自带样式，并且每一款浏览器所自带的样式不同，为统一浏览器样式，需要对浏览器的默认样式进行清除

在设置导航条的时候，如果所需的导航条是横向的，则需要将元素设置为浮动

可以将公共的样式重写到另外一个单独的选择器中，简化代码

需要水平放置在同一行的需要对放置在同一行的所有元素都设置为水平浮动

往往父元素的高度是不能显性赋值的，避免当内容过多时导致内容溢出等异常情况

行内块元素不适合作为外部元素

若元素上方有浮动元素，其下方的元素会自动上移，而文字不会上移，文字会自动环绕在浮动元素之上

伪元素是css内置好的而不能进行自定义，只能修改和调用定义好的伪元素

浮动会使得元素脱离文档流，造成高度塌陷的问题

相对定位的位置是相对于其他元素之外的，可以理解成浮在其他元素之上，相对定位理解成“灵魂出窍”，虽然元素的确发生了移动，但是该元素浮在其他元素之上，而其之前的“身体”仍然占据着位置

只要position的值不是static即为开启了定位

粘滞定位由于其兼容性的影响，往往不常使用

对于开启了定位的元素，并且和绝对定位或者相对定位等设置无关，可以通过z-index属性来指定元素的层级，index需要一个整数作为参数，且值越大表示的是元素的层级越高，元素的层级越高越优先显示，当该参数的值大小相同时，根据HTML代码中出现的顺序来决定元素的层级，祖先元素的层级再高也不会盖住后代元素

字体文件本身是一种矢量图，无论如何进行放大缩小都不会失真

常见使用的图标字体库是现成的，不需要通过自己创建

all.min.css和all.css的区别在于前者是压缩过的，前者的大小相对而言更小一点

使用字符实体进行设置图标字体时注意得先通过link标签来引入相关css文件

注意父元素浮动和子元素浮动是有区别的，同时注意可以设置让父元素向左浮动，让子元素向右浮动，呈现的效果是整体向左移，在父元素的格子里面的子元素整体向右移

为了避免子元素高度的改变导致父元素高度的变化，可以将父元素的高度显式赋值，避免其高度受影响

使用overflow属性来规定当内容溢出元素框时的处理方式

简写属性是不存在顺序要求的（写一大串就可以）

创建一个组件的时候注意首先创建一个对整个组件进行作用的外部容器

组件的大小需要依据其包含的具体内容确定时，不能设置其具体的大小

由于图片是存放在容器里面的，所以图片本身也需要使用容器包含其中，同理文字也需要使用专门的容器进行存放

给元素的width属性进行赋值时，如果单位是%则表示其定义基于包含块（父元素）宽度的百分比宽度

在网页也可以使用表单，网页中的表单用于将本地的数据提交给远程的服务器

向右或者向左浮动的元素的排序取决于在html中代码出现的先后顺序
