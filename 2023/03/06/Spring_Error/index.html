通过Spring Initializr构建SpringBoot项目没有自动设置成spring项目：

  • 将对应的java、resources文件夹右键点击mark as …
  • 将对应的pom.xml文件右键点击Add as maven project …

使用maven导包一直导入不进来：

在导入行显示进行了导入但是使用对应类的时候仍然显示没有import包时报程序包不存在的问题

  • 清除 Idea 缓存，很多情况下idea导致的问题通过这个方式基本都能解决
  • 方式：-> 工具栏 -> File -> Invalidate Caches / Restart

报错expected single matching bean but found 2：

  • @MapperScan(“com.hmdp.mapper”)中限定的范围不小心改成了@MapperScan(“com.hmdp”)

使用maven导包一直显示waiting for maven import completion：

  • 将maven版本由3.8.5降为3.5.4
  • idea对于不同的maven适配不同，需要进行相应适配
    • IDEA 2022 兼容maven 3.8.1及之前的所用版本
    • IDEA 2021 兼容maven 3.8.1及之前的所用版本
    • IDEA 2020 兼容Maven 3.6.3及之前所有版本
    • IDEA 2018 兼容Maven3.6.1及之前所有版本
    • IDEA各版本对Maven版本兼容

显示xml不对并且一直无法导入：

  • 配置镜像源的时候多打了一列标签导致识别错误
  • 显示某个依赖包导入不成功可以将其在仓库中手动删去，然后在idea中清除缓存重新导入

测试类中使用@Autowired无法将对应的服务类进行导入报错空指针：

Maven相关配置：

  • Use settings from .mvn/maven.config：表示使用后面的配置文件来覆盖原来的设置

报错Unsupported class file major version 60：

配置好了Redis相关的属性但是依旧报错：

  • 配置Redis的文档不正确

  • 由于yaml是根据缩进进行控制的，所以可能是缩进没有配置好

  • 配置Redis相关的xml配置缺漏

使用mybatis-Plus过程中正常调用方法但是显示SQL语法错误

  • 错误原因：使用了order作为表名，但是order本身就是SQL中的关键导致报错
  • 解决方法：修改数据库表名，然后使用mybatisPlus的注解手动映射即可 @TableName(“my_order”)

使用SQL文件运行后数据库依然错误：

  • 错误原因：由navicat生成的sql文件只会产生新增加的表，但不会自动删除原本SQL文件中不存在的表
  • 解决方法：清空数据库再运行SQL文件

慢慢配置了Redis的对应配置文件的密码但是报错ERR AUTH called without anypassword configured for the default user

  • 错误原因：配置没有生效，所以导致其实并没有给docker容器设置对应密码

  • 解决方法：在yml中配置

给某个实体类设置了null值但是实际运行过程发现可以对实体赋值正常数值，但是不能设null

  • 错误原因：

    • 使用Mybatis-plus操作数据库时，如果未自己写sql而是采用Mybatis-plus的方法设置sql的情况下，默认Mybatis-plus生成sql时会将设置为null的字段忽略掉(不更新该字段)；所以默认情况下，使用Mybatis-plus的方法操作数据库时没办法设置一个字段的值为null(除非该字段数据库表设置默认值就为null)
    • Mybatis-plus设计如此很好理解，比如现在我们的系统中一般都是做逻辑删除即修改删除标记的字段为删除状态的值，此时执行update语句时，一般根据id作为条件只设置删除标记字段的值就行，其他字段值不做设置也就是为null，如果Mybatis-plus也将这些字段进行更新的话，那么其他字段全是null，这样会造成数据丢失，问题很大；如果每个字段全部设置成之前的值，需要先进行一次查询操作，以及设置每个字段的值的操作，比较麻烦一些，很明显没有必要；所以Mybatis-plus默认忽略字段值为null的字段不做更新

  • 解决方法：

    • 手动写sql语句

      • 在mapper.xml中写对应的sql语句(insert语句/update语句/where条件中设置对应字段的值为null即可)

    • 实体类对应字段添加注解

报错400 ：JSON parse error: Cannot deserialize instance of java.lang.Integer out of VALUE_TRUE token：

  • 错误原因：JSON解析错误，前端传过来的值与后端值类型不匹配，例如前端传一个字符串，结果使用的是int接受
  • 解决方法：修改对应的数值

下载 jetbrain 系的IDE无法正常使用破解软件进行激活

  • 错误原因：自带的软件破解方式只能破解2021年的软件而不能破解2022年的软件版本
  • 解决方法：重新下载前一年的版本

无法连接上数据库

  • 错误原因：使用的Linux 上的mysql是使用docker进行搭建的，所以忘记了密码
  • 解决方法：在配置文件中增加 skip-grant-tables 跳过密码校验进行登录

编写的配置类一直无法生效

  • 错误原因：config包未放在同一目录下，没有仔细放置
  • 解决方法：放到同一目录下

在使用Spring框架中的Config配置中心时，设置了配置文件并commit到了仓库但依然无法通过http获取

  • 错误原因：java.nio.charset.MalformedInputException: Input length = 1， 是因为你的配置文件里面有中文或者是你的编码格式不正确导致
  • 解决方法：使用英语或者使用UTF-8编码集

Idea配置cmd脚本时，报错open Local Terminal_Failed to start [powershell.exe]

  • 错误原因：idea找不到对应的cmd.exe文件
  • 解决方法：setting->tool->终端，设置对应的cmd.exe即可

无法通过idea ssh连接Ubuntu虚拟机

  • 错误原因：缺少相关设置
  • 解决方法：安装OpenSSH，修改OpenSSH相关配置文件

使用nacos配置中心的时候无法读取对应的设置好的配置（尚未解决）

  • 错误原因：没有在pom.xml中导入对应的依赖（有一次导入错误，导入到了父项目的xml中）
  • 解决方法：在对应项目中导入nacos相关依赖即可
  • 提示：当配置 yml 时，如果部分属性出现警告或者没有自动补全，说明很可能是没有引入对应的依赖

使用linux无法登陆到root用户

  • 错误原因：部分系统默认不开启root用户登陆，需要手动打开并且设置 root用户的密码
  • 解决方法：sudo passwd root 修改密码即可

使用Xshell有时运行指令速度较慢：

  • 解决方法：在ssh配置文件中关闭DNS 解析即可

在对微服务项目进行打包时发现报错：dependencies.dependency.[com.xiaozhao:commons:0.0.1-SNAPSHOT] for com.xiaozhao:commons:0.0.1-SNAPSHOT is referencing itself. @

  • 错误原因：父项目本身只是个框架，却错误的在pom.xml中增加了对common模块的依赖
  • 解决方法：删除对common模块的依赖即可
